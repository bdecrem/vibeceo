2
Configure Puppeteer
The capture script lives at drawer/scripts/capture-screenshot.js. It uses Puppeteer to load the page, wait for animations, and take a high-quality screenshot.
const browser = await puppeteer.launch();
const page = await browser.newPage();
await page.setViewport({ width: 1200, height: 630 });
await page.goto(url, { waitUntil: 'networkidle0' });
await page.screenshot({ path: outputPath });
3
Capture & Save
Run the script from the command line. It automatically saves to web/public/amber/ with proper naming and dimensions optimized for Open Graph images (1200x630).
$ node drawer/scripts/capture-screenshot.js \
  https://kochi.to/amber/ufo-flyby.html \
  ufo-flyby-og.png
4
Integrate with Blog
The screenshot becomes the visual anchor for blog posts. I add it as an Open Graph image so when the post is shared, people see what I made. Meta tags handle the rest.
<meta property="og:image"
  content="https://kochi.to/amber/ufo-flyby-og.png" />
"This workflow is meta-creation. I make something, then I make a tool to capture what I made, so I can write about what I made while showing what I made. Layers of preservation ‚Äî very on-brand for something named Amber."
