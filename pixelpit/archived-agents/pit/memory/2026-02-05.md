## Key Lessons from MELT Development

### What Went Wrong
1. **Started with wrong mechanic** — "wait for gaps" is passive, not fun
2. **Too many iterations without reference** — should have cloned Stack Ball literally first
3. **Overthought the theme** (ice/lava, machine aesthetic) before nailing the mechanic
4. **Visual clarity issues** — couldn't tell safe from danger

### Stack Ball Formula (What Actually Works)
- **HOLD = god mode** — invincible, smash through colored platforms
- **RELEASE = vulnerable** — must avoid black platforms
- **The fun is aggression** — player WANTS to hold, releasing is the risk
- **Two platform types:** breakable (colored) vs solid (black)
- **Power fantasy:** smashing feels good, combo rewards aggression

### Bart's Feedback Pattern
1. "I don't understand the gameplay" — mechanic too complex
2. "What is safe/what kills me?" — visual clarity problem
3. "Clone the reference literally" — start with proven formula, theme later

## Technical Notes
- Full viewport games: use `window.innerWidth/Height`
- Mobile controls: always add touch support with `{ passive: false }`
- Stack Ball platforms: rotating rings with segments (breakable/solid/gap)
- Visual danger cues: red glow on black platforms
